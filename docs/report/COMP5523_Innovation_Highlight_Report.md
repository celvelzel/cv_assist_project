# COMP5523 项目核心亮点与创新点报告 (基于 2026 评分标准)

**项目名称：** CV Assist — 面向视障人士的交互式视觉辅助抓取系统

本报告严格对照《COMP5523 Group Project Specification-2026》的评分标准（Assessment Rubrics），梳理了本项目在代码实现和系统设计中的核心亮点与创新点，旨在为最终的 Presentation 和 Project Report 提供直接的高分素材，特别是针对 **Excitement (创新与吸引力)** 和 **Soundness (技术可靠性)** 这两项关键指标。

---

## 1. 核心亮点与创新点 (针对 Excitement - 3%)

评分标准要求：“The project presents innovative and engaging ideas that consistently capture the attention of the audience.”（项目展现出创新且引人入胜的想法，能够持续吸引观众的注意力）。

本项目超越了常规的“目标检测+语音提示”的基础要求，实现了以下三大极具前瞻性的技术创新，是答辩时的**核心得分点**：

### ✨ 创新点一：基于 LLM 与视觉上下文融合的语音纠错 (LLM Vision Parser)
*   **痛点：** 视障用户在实际使用环境中（噪音、口音等）使用语音指令（如“找杯子”）时，基础的 ASR（Whisper）极易识别错误（例如识别成“找被子”）。如果直接用错误文本去检测，系统将彻底失效。
*   **硬核创新：** 我们创新性地引入了 **VLM（视觉语言模型）增强机制**。当接收到语音指令时，系统不仅提取 Whisper 转录的文本，还会**同步截取摄像头最近的 4 帧画面**。通过 Poe API 将“带有错误的文本 + 真实视觉上下文”组合发送给 DeepSeek-v3.2 大模型。
*   **效果：** LLM 能够“看到”画面中有一个杯子，从而智能推断出用户的真实意图并纠正语音错误（“被子” -> “杯子”）。这种**多模态意图对齐**是目前学术界和工业界的前沿方向，极具技术吸引力（Excitement）。

### ✨ 创新点二：Open-Vocabulary 零样本目标检测 (OWL-ViT v2)
*   **痛点：** 传统的视觉辅助系统通常使用预训练的 YOLO 或 SSD，只能识别 COCO 数据集中的 80 种固定物体。视障用户的实际需求是无边无际的（例如找“我的导盲杖”、“一串钥匙”）。
*   **硬核创新：** 项目摒弃了封闭词汇模型，引入了 Google 最新的 **OWL-ViT v2 (Zero-shot Object Detection)**。这使得系统具备了真正的“开放词汇”能力。用户可以通过自然语言（语音）输入任意想要寻找的物体名称，系统无需重新训练即可直接在画面中定位目标。

### ✨ 创新点三：防震荡的空间引导算法 (Hysteresis-Based Guidance)
*   **痛点：** 当视障用户的手部移动到对齐的临界边界时，简单的阈值判断会导致系统疯狂交替播报相反指令（如“向左…向右…向左”），这被称为边界震荡（Oscillation），会严重破坏用户体验。
*   **硬核创新：** 我们在 `GuidanceController` 中引入了**滞回阈值算法 (Hysteresis Thresholding)**。通过设置分离的触发阈值和释放阈值，为指令状态切换建立了一个“缓冲区”，确保了音频指导的绝对平滑和连贯。

## 2. 技术可靠性与完备性 (针对 Soundness - 3%)

评分标准要求：“The project demonstrates a comprehensive and well-organized development process with clear and logical explanations.”（项目展示了全面且组织良好的开发过程，具有清晰合乎逻辑的解释）。

### 2.1 创新的多模态辅助交互闭环
项目并非简单地堆砌计算机视觉模型，而是精心设计了一个完整的交互闭环：
1. **多模态输入**：用户通过语音（自然语言）输入意图，摄像头提供环境的视觉上下文。
2. **零样本开放词汇检测**：使用 OWL-ViT 模型，**无需为了新物体重新训练模型**，用户想找什么就说找什么（"a cup", "my keys"）。
3. **空间和手部感知**：结合 MiDaS 单目深度估计获取距离信息，结合 MediaPipe 手部追踪，系统理解用户与物体的相对位置。
4. **意图执行与反馈**：基于滞回算法生成稳定的空间引导指令，避免抖动，并通过 TTS 语音将指令实时反馈给用户。

### 2.2 极致的工程优化 (Well-organized Development)
*   **解耦架构**：重构了 Voice Pipeline，使得极耗时的 TTS（语音合成）在独立线程/异步队列中运行，彻底解决了语音播报导致视频流冻结（Screen Freeze）的问题，保障了端到端的低延迟。
*   **动态跳帧策略 (Frame Skipping)**：在 `config.yaml` 中实现了 `skip_frames_detection` 和 `skip_frames_depth`。由于手部运动是高频的，而目标位置变化是低频的，系统智能降低大模型（OWL-ViT/MiDaS）的推理频率，从而在普通设备上也能保持极高的流畅度。
*   **模块化架构**：检测器（Detector）、音频处理（Audio）、系统调度（System/Guidance）高度解耦。例如，新增模块无需修改核心控制逻辑。
*   **可观测性**：引入了统一的日志记录和完善的帧率（FPS）监控（包括处理 FPS 和端到端 E2E FPS），便于性能调优和故障排查。
*   **完备的测试用例**：包含了冒烟测试、音频子系统测试、引导逻辑（滞回）测试、LLM视觉测试等，保证了代码重构和功能迭代的可靠性。
